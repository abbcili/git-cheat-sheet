## Установка Git

GitHub предоставляет оконное приложение с графическим интерфейсом для выполнения основных операций с репозиторием, и консольную версию Git с автоматическими обновлениями для расширенных сценариев работы.


### GitHub Desktop

[desktop.github.com](https://desktop.github.com/)

Дистрибутивы Git для систем Linux и POSIX доступны на официальном сайте Git SCM.


### Git для всех платформ

[git-scm.com](git-scm.com)

## Первоначальная настройка

__Настройка информации о пользователе для всех локальных репозиториев__

_`$ git config --global user.name "[имя]"`_

:point_up: Устанавливает имя, которое будет отображаться в поле автора у выполняемых вами коммитов 


_`$ git config --global user.email "[адрес электронной почты]"`_

:point_up: Устанавливает адрес электронной почты, который будет отображаться в информации о выполняемых вами коммитах 

___

## Создание репозитория
 
Создание нового репозитория или получение его по существующему URL-адресу

_`$ git init [название проекта]`_

:point_up: Создаёт новый локальный репозиторий с заданным именем 


_`$ git clone [url-адрес]`_

:point_up: Скачивает репозиторий вместе со всей его историей изменений 

___

## Внесение изменений

__Просмотр изменений и создание коммитов (фиксация изменений)__

_`$ git status`_

:arrow_up_small: Перечисляет все новые или изменённые файлы, которые нуждаются в фиксации 


_`$ git diff`_

:arrow_up_small: Показывает различия по внесённым изменениям в ещё не проиндексированных 


_`$ git add [файл]`_

:arrow_up_small: Индексирует указанный файл для последующего коммита 


_`$ git diff --staged`_

:arrow_up_small: Показывает различия между проиндексированной и последней зафиксированной версиями файлов 

_`$ git reset [файл]`_

:arrow_up_small: Отменяет индексацию указанного файла, при этом сохраняет его содержимое 


_`$ git commit -m "[сообщение с описанием]"`_

:arrow_up_small: Фиксирует проиндексированные изменения и сохраняет их в историю версий 

___

## Коллективная работа

__Именованные серии коммитов и соединение результатов работы__

_`$ git branch`_

:small_red_triangle: Список именованных веток коммитов с указанием выбранной ветки 


_`$ git branch [имя ветки]`_

:small_red_triangle: Создаёт новую ветку 


_`$ git checkout [имя ветки]`_

:small_red_triangle: Переключается на выбранную ветку и обновляет рабочую директорию до её состояния 


_`$ git merge [имя ветки]`_

:small_red_triangle: Вносит изменения указанной ветки в текущую ветку 


_`$ git branch -d [имя ветки]`_

:small_red_triangle: Удаляет выбранную ветку 

___

## Операции с файлами

__Перемещение и удаление версий файлов репозитория__

_`$ git rm [файл]`_

:arrow_up: Удаляет конкретный файл из рабочей директории и индексирует его удаление 


_`$ git rm --cached [файл]`_

:arrow_up: Убирает конкретный файл из контроля версий, но физически оставляет его на своём месте 


_`$ git mv [оригинальный файл] [новое имя]`_

:arrow_up: Перемещает и переименовывает указанный файл, сразу индексируя его для последующего коммита 

___

## Игнорирование некоторых файлов

__Исключение временных и вторичных файлов и директорий__

`*.log  build/  temp-*`

Git будет игнорировать файлы и директории, перечисленные в файле .gitignore с помощью wildcard синтаксиса


_`$ git ls-files --others --ignored --exclude-standard`_

:arrow_up: Список всех игнорируемых файлов в текущем проекте 

___

## Сохранение фрагментов

__Сохранение и восстановление незавершённых изменений__

_`$ git stash`_

Временно сохраняет все незафиксированные изменения отслеживаемых файло


_`$ git stash pop`_

:point_up_2: Восстанавливает состояние ранее сохранённых версий файлов


_`$ git stash list`_

:point_up_2: Выводит список всех временных сохранений


_`$ git stash drop`_

:point_up_2: Сбрасывает последние временно сохранённыe изменения

___

## Просмотр истории

__Просмотр и изучение истории изменений файлов проекта__

_`$ git log`_

:top: История коммитов для текущей ветки 


_`$ git log --follow [файл]`_

:top: История изменений конкретного файла, включая его переименование 


_`$ git diff [первая ветка]...[вторая ветка]`_

:top: Показывает разницу между содержанием коммитов двух веток 


_`$ git show [коммит]`_

:top: Выводит информацию и показывает изменения в выбранном коммите 

___

## Откат коммитов

__Удаление ошибок и корректировка созданной истории__

_`$ git reset [коммит]`_

:point_up_2: Отменяет все коммиты после заданного, оставляя все изменения в рабочей директории 


_`$ git reset --hard [коммит]`_

:point_up_2: Сбрасывает всю историю вместе с состоянием рабочей директории до указанного коммита. 

___

## Синхронизация с удалённым репозиторием

__Регистрация удалённого репозитория и обмен изменениями__

_`$ git fetch [удалённый репозиторий]`_

:point_up: Скачивает всю историю из удалённого репозитория 


_`$ git merge [удалённый репозиторий]/[ветка]`_

:point_up: Вносит изменения из ветки удалённого репозитория в текущую ветку  локального репозитория 


_`$ git push [удалённый репозиторий] [ветка]`_

:point_up: Загружает все изменения локальной ветки в удалённый репозиторий 


_`$ git pull`_

:point_up: Загружает историю из удалённого репозитория и объединяет её с локальной. pull = fetch + merge
